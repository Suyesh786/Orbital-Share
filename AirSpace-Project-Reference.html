§1 — The Problem & The Solution
Why this migration was done
🚫
The Problem The old app bundle identifier com.neardrop.app was silently blacklisted/quarantined by macOS security policies because it used ad-hoc signing profiles. macOS was blocking all local network access without ever showing a permission prompt — making the app unable to discover or connect to nearby devices on the LAN. There was no way to fix this without a completely fresh identity.
✅
The Solution A brand-new Electron app was created at /Users/suyesh/Desktop/Airspace with a pristine App ID com.airspace.desktop. When macOS sees this new ID for the first time, it will show a fresh Local Network permission prompt. The full frontend UI, animations, and logic were migrated verbatim — only the identity strings were changed.

§6 — Key Code Written — Exact Quotes
The most important lines in the new codebase

6.1 — Identity Constants (electron/main.cjs lines 22–28)

These 6 lines are the heart of the entire migration. They define the new clean identity:

// The Bonjour service type — advertised/browsed on the LAN via native dnssd
const BONJOUR_SERVICE_TYPE = "airspace";      // → publishes _airspace._tcp

const APP_NAME = "AirSpace";                  // → tray menus, window title, notifications

const TRAY_POPUP_ARG = "--airspace-tray-popup";
const HIDDEN_RUNTIME_ARG = "--airspace-hidden-runtime";
const TRAY_GUID = "a1b2c3d4-e5f6-7890-abcd-ef1234567890";  // fresh GUID

6.2 — Native dnssd Advertisement (Receiver Mode)

When a user enables Receiver mode, this code publishes the device on the LAN via native C++:

// Native dnssd.Advertisement — registers _airspace._tcp on the LAN
publishedReceiverAd = new dnssd.Advertisement(
  dnssd.tcp(BONJOUR_SERVICE_TYPE),  // → _airspace._tcp
  LOCAL_WEBSOCKET_PORT,             // → port 8080
  {
    name: `${APP_NAME} ${deviceId}`,
    txt: {
      available: "1",
      deviceId,
      username,
      deviceType,
      platform,
    },
  }
);
publishedReceiverAd.start();

6.3 — Native dnssd Browser (Sender/Discovery Mode)

When the user opens Discovery mode, this scans the LAN for other AirSpace instances:

// Native dnssd.Browser — bypasses JS-layer Bonjour blocks
dnssdBrowser = new dnssd.Browser(dnssd.tcp(BONJOUR_SERVICE_TYPE));

dnssdBrowser.on("serviceUp", (service) => {
  // ← CRITICAL: alias fullname → fqdn so the frontend Map key works
  upsertLanReceiver({ ...service, fqdn: service.fullname });
});

dnssdBrowser.on("serviceDown", (service) => {
  removeLanReceiver({ ...service, fqdn: service.fullname });
});

dnssdBrowser.start();

All IPC channel names kept as orbitalshare: — they are internal strings, NOT network identifiers:

// All channel names preserved exactly — frontend depends on these strings

6.5 — Lifecycle Cleanup (Resource Management)

Properly unregisters all native dnssd instances on quit:

app.on("before-quit", () => {
  isQuitting = true;
  stopPublishedReceiverService();  // stops Advertisement
  stopBonjourDiscovery();           // stops Browser
  stopLanReceiverSweep();           // clears stale-receiver timer
  localNetworkingStarted = false;
  trayWindow?.destroy();
  tray?.destroy();
});

§9 — How to Verify It Works
Step-by-step testing protocol
  • ①
    Launch the app
    open /Users/suyesh/Desktop/Airspace/release/mac-arm64/AirSpace.app
    macOS should show a fresh Local Network permission prompt for "AirSpace". Accept it.
  • ②
    Check System Settings
    Go to System Settings → Privacy & Security → Local Network.
    You should see "AirSpace" in the list (not NearDrop).
  • ③
    Verify Bonjour advertising (Receiver Mode)
    Enable Receiver mode in the app, then run in Terminal:
    dns-sd -B _airspace._tcp local
    Should show your device advertising as _airspace._tcp.
  • ④
    Two-device test
    Run AirSpace on two Macs on the same WiFi network. One in Receiver mode, one in Sender/Discovery mode. The sender should discover the receiver on the LAN.
  • ⑤
    Confirm old identity is gone
    The old com.neardrop.app entry should no longer appear in the Local Network permission list after a fresh macOS restart.
